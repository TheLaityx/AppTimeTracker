## 1. 环境准备

### 1.1 系统要求
- Windows 10 / 11 (64位)
- 至少 4GB 内存（Rust 编译需要较多内存）
- 至少 8GB 空闲磁盘（工具链较大，但最终产物很小）

### 1.2 必须安装的工具

| 工具 | 说明 | 是否必须 |
|------|------|--------|
| Rust | 核心编译器 | ✅ 必须 |
| Node.js 18+ | 前端构建工具 | ✅ 必须 |
| Visual Studio Build Tools | Windows C++ 编译支持 | ✅ 必须 |
| NSIS | 生成安装包 | ✅ 必须 |
| WebView2 Runtime | Tauri 需要（Win11 已内置） | ✅ 必须 |

## 2. 安装依赖工具

### 步骤 1：安装 Visual Studio Build Tools

1. 打开 https://visualstudio.microsoft.com/visual-cpp-build-tools/
2. 下载 **Build Tools for Visual Studio 2022**
3. 安装时勾选：
   - ✅ **C++ 生成工具**（必须）
   - ✅ Windows 10/11 SDK
4. 安装完成后**重启电脑**

### 步骤 2：安装 Rust

打开 PowerShell（以管理员运行），执行：

```powershell
# 下载并运行 Rust 安装器
Invoke-WebRequest -Uri "https://win.rustup.rs/x86_64" -OutFile "rustup-init.exe"
.\rustup-init.exe
```

安装时选择 **默认安装（1）**，等待完成。

验证安装：
```powershell
rustc --version
# 应显示: rustc 1.78.0 (xxx)

cargo --version
# 应显示: cargo 1.78.0
```

### 步骤 3：安装 Node.js

1. 打开 https://nodejs.org/
2. 下载 **LTS 版本**（如 v20.x）
3. 运行安装包，全部默认安装

验证：
```powershell
node --version  # v20.x.x
npm --version   # 10.x.x
```

### 步骤 4：安装 NSIS（打包安装程序用）

1. 打开 https://nsis.sourceforge.io/Download
2. 下载并安装 **NSIS 3.x**（默认路径 `C:\Program Files (x86)\NSIS`）
3. 安装完成后无需额外配置

### 步骤 5：验证 WebView2（Win11 已预装）

Win11 通常已内置 WebView2，如未安装：
1. 打开 https://developer.microsoft.com/zh-cn/microsoft-edge/webview2/
2. 下载 **Evergreen 独立安装包**并安装

## 3. 配置项目

### 步骤 1：下载/解压项目

将项目文件夹（`AppTimeTracker`）放到任意位置，例如：
```
C:\Projects\AppTimeTracker\
```

### 步骤 2：安装 npm 依赖

打开 **命令提示符** 或 **PowerShell**，进入项目根目录：

```powershell
cd C:\Projects\AppTimeTracker

# 安装前端依赖（仅用于编译，不会打入最终包）
npm install
```

### 步骤 3：安装 Tauri CLI

```powershell
npm install -g @tauri-apps/cli@^2
```

验证：
```powershell
tauri --version
# 应显示: tauri-cli 2.x.x
```

## 4. 生成图标

图标需要放在 `src-tauri/icons/` 目录下。

### 方法一：自动生成（推荐）

准备一张 **1024x1024 的 PNG 图片**（你的应用图标），命名为 `app-icon.png`，放在项目根目录，然后运行：

```powershell
cd C:\Projects\AppTimeTracker
npx tauri icon app-icon.png
```

这会自动在 `src-tauri/icons/` 生成所有需要的图标文件。

### 方法二：使用提供的默认图标

项目已包含一套默认 SVG 图标的生成脚本。运行：

```powershell
node scripts/gen-icons.js
```

## 5. 编译发布版本

### 开发模式预览（可选）

在编译前可先预览效果：

```powershell
cd C:\Projects\AppTimeTracker
npm run tauri dev
```

> ⚠️ 开发模式会显示窗口，不代表最终行为。

### 编译正式发布包

```powershell
cd C:\Projects\AppTimeTracker
npm run tauri build
```

**首次编译说明：**
- Rust 会下载并编译所有依赖，**大约需要 10~20 分钟**
- 后续编译只需 1~3 分钟（增量编译）
- 期间 CPU 会满载，属于正常现象

**编译完成后，产物位于：**
```
src-tauri/target/release/
├── app-time-tracker.exe          ← 可直接运行的 EXE（约 8MB）
└── bundle/
    └── nsis/
        └── 应用时长_1.0.0_x64-setup.exe  ← 安装包（约 6MB）
```

## 6. 打包安装程序

编译时已自动生成 NSIS 安装包，无需额外操作。

### 安装包功能（自动包含）：

✅ 用户可选择安装位置  
✅ 可选"创建桌面快捷方式"  
✅ 可选"创建开始菜单快捷方式"  
✅ 自动注册开机启动（用户可在设置中关闭）  
✅ 支持卸载（控制面板 → 程序和功能）  
✅ 界面语言：简体中文  

### 自定义安装包（可选）

如需自定义安装包外观，编辑 `src-tauri/tauri.conf.json` 中的 `bundle.windows.nsis` 部分：

```json
"nsis": {
  "languages": ["SimpChinese"],
  "displayLanguageSelector": false,
  "installMode": "perMachine",
  "shortcutsEnabled": true,
  "license": "LICENSE.txt"
}
```

## 7. 安装包功能说明

### 用户首次安装流程

```
运行安装包
    ↓
选择安装位置（默认 C:\Program Files\应用时长）
    ↓
勾选选项：
  ☑ 创建桌面快捷方式
  ☑ 开机自动启动（后台，不显示窗口）
    ↓
安装完成
    ↓
首次启动 → 显示欢迎/隐私说明界面 → 用户确认
    ↓
关闭窗口 → 隐藏到系统托盘（右下角）
    ↓
此后开机自动后台运行，不显示窗口
```

### 托盘图标操作

| 操作 | 效果 |
|------|------|
| 左键单击 | 切换显示/隐藏主窗口 |
| 右键菜单 → 打开主界面 | 显示主窗口并切换到首页 |
| 右键菜单 → 设置 | 显示设置页面 |
| 右键菜单 → 退出应用 | 彻底关闭程序 |

### 关闭行为

- **点击窗口 × 按钮** → 隐藏到托盘，继续后台追踪
- **右键托盘 → 退出应用** → 彻底退出，停止追踪

## 8. 常见问题

### ❌ 错误：`error: linker 'link.exe' not found`

**解决：** 未安装 Visual Studio Build Tools 或未重启电脑。按步骤 1 重新安装后重启。

### ❌ 错误：`NSIS not found`

**解决：** 
1. 确认 NSIS 已安装
2. 将 NSIS 添加到环境变量：
   ```
   C:\Program Files (x86)\NSIS
   ```
   或重新安装 NSIS 后重启 PowerShell

### ❌ 编译报 Windows API 找不到

**解决：** 确认安装了正确版本的 Windows SDK。在 VS Build Tools 安装器中，勾选 Windows 11 SDK。

### ❌ `npm install` 很慢

**解决：** 设置 npm 镜像：
```powershell
npm config set registry https://registry.npmmirror.com
```

Rust 编译加速（国内镜像），在 `%USERPROFILE%\.cargo\config.toml` 写入：
```toml
[source.crates-io]
replace-with = "rsproxy"

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true
```

### ❌ 运行后没有托盘图标

**解决：** 检查 `src-tauri/icons/` 目录下是否有 `icon.png`，确保图标文件存在再重新编译。

### ❌ 窗口显示为黑色

**解决：** WebView2 Runtime 未安装或版本过旧，请更新。
